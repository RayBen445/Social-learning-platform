## System Accounts

### Bot Account
Used for automated tasks, announcements, and system functions.

```
Email:    bot@learnloop.app
Role:     System Bot
Badge:    Bot Verification Badge
Purpose:  Automated announcements, system notifications, scheduled tasks
Status:   Created via setup script
```

**Password:** 
- Generated by: `pnpm run setup-credentials`
- Stored in: `.admin-credentials.json`
- Never shared publicly

### Admin Account  
Used for platform administration and content moderation.

```
Email:    admin@learnloop.app
Role:     Administrator
Badge:    System Verification Badge
Purpose:  Content moderation, user management, system settings
Status:   Created via setup script
Access:   /admin/moderation dashboard
```

**Password:**
- Generated by: `pnpm run setup-credentials`
- Stored in: `.admin-credentials.json`
- Never shared publicly

## User IDs (from Supabase Auth)

After creating the accounts in Supabase, add these UUIDs to your environment:

```env
# In .env.local or Vercel deployment settings
NEXT_PUBLIC_BOT_USER_ID=00000000-0000-0000-0000-000000000001
NEXT_PUBLIC_ADMIN_USER_ID=00000000-0000-0000-0000-000000000002
```

**Where to find:**
1. Go to Supabase Dashboard
2. Select your project
3. Go to Authentication > Users
4. Copy the UUID for each user

## Supabase Credentials

Get these from Supabase Dashboard > Settings > API:

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-here
```

⚠️ **Important:**
- Never expose SERVICE_ROLE_KEY in client code
- Use only ANON_KEY for client-side requests
- Keep all keys in environment variables

## Vercel Environment Variables

When deploying to Vercel, add these in:
**Settings > Environment Variables**

```
# Database
NEXT_PUBLIC_SUPABASE_URL = [your-supabase-url]
NEXT_PUBLIC_SUPABASE_ANON_KEY = [your-anon-key]
SUPABASE_SERVICE_ROLE_KEY = [your-service-role-key]

# System Accounts  
NEXT_PUBLIC_BOT_EMAIL = bot@learnloop.app
BOT_PASSWORD = [generated-bot-password]
NEXT_PUBLIC_ADMIN_EMAIL = admin@learnloop.app
ADMIN_PASSWORD = [generated-admin-password]
NEXT_PUBLIC_BOT_USER_ID = [bot-user-uuid]
NEXT_PUBLIC_ADMIN_USER_ID = [admin-user-uuid]

# Email (if using external service)
SENDGRID_API_KEY = [optional-sendgrid-key]
NEXT_PUBLIC_MAIL_FROM = noreply@learnloop.app

# URLs
NEXT_PUBLIC_PROD_SUPABASE_REDIRECT_URL = https://your-domain.com/auth/callback
```

Mark sensitive values as "Sensitive" in Vercel.

## Account Creation Steps

### 1. Generate Credentials
```bash
pnpm run setup-credentials
```

Saves to `.admin-credentials.json` with passwords.

### 2. Create in Supabase Auth

Go to Supabase Dashboard > Authentication > Users > Add user

**For Bot Account:**
- Email: `bot@learnloop.app`
- Password: [from .admin-credentials.json]
- Auto Confirm: ✓ Enable
- Click "Create user"

**For Admin Account:**
- Email: `admin@learnloop.app`
- Password: [from .admin-credentials.json]
- Auto Confirm: ✓ Enable
- Click "Create user"

### 3. Copy User IDs

In the users list, copy the UUID for each user.

### 4. Seed Profiles

Edit `scripts/008_seed_admin_bot_users.sql`:
- Replace `00000000-0000-0000-0000-000000000001` with bot user ID
- Replace `00000000-0000-0000-0000-000000000002` with admin user ID

Run in Supabase SQL Editor.

### 5. Update Environment

Add to `.env.local`:
```env
NEXT_PUBLIC_BOT_USER_ID=[bot-uuid]
NEXT_PUBLIC_ADMIN_USER_ID=[admin-uuid]
```

## Security Best Practices

✅ **DO:**
- Store passwords in secure password manager
- Add env vars to Vercel (not in code)
- Rotate passwords periodically (monthly or on breach)
- Keep .admin-credentials.json secure (add to .gitignore)
- Use different passwords for different environments
- Enable 2FA for Supabase and Vercel accounts

❌ **DON'T:**
- Commit credentials to GitHub
- Share passwords in Slack/email
- Use same passwords for dev and production
- Expose SERVICE_ROLE_KEY in client code
- Leave credentials in browser local storage
- Log sensitive data to console

## Rotating Passwords

If a password is compromised:

### 1. Change in Supabase
1. Go to Supabase Dashboard > Authentication > Users
2. Click the user
3. Click "Reset password" 
4. User gets reset email

### 2. Update Environment Variables
1. Update in `.env.local`
2. Update in Vercel Settings > Environment Variables
3. Redeploy application

### 3. Update Password Manager
- Update stored credentials
- Notify team of new passwords
